Case Study 01

Sarvar Home Server

Problem
I wanted a practical self-hosted environment for learning Linux, networking, and service management without relying on disposable demo setups.
What I Built
A hands-on homelab server running multiple Docker services for media, DNS filtering, reverse proxying, and remote access.
Tools Used
Ubuntu Server, Docker, Tailscale, Pi-hole, Immich, Nginx Proxy Manager, GitHub Pages, monitoring and documentation.
Security Consideration
Kept services private where possible, used secure remote access, avoided exposing internal endpoints, and separated access paths logically.
Outcome
A self-hosted infrastructure platform that demonstrates systems thinking, service ownership, and responsible access design.
What I Learned
How to troubleshoot services, manage containerized workloads, think about exposure, and maintain a useful lab instead of a fragile demo.
Case Study 02

Hermes Agent AI Automation

Problem
I wanted an AI assistant that could do more than chat — it needed to support real workflows, communication, and knowledge management.
What I Built
A Telegram-connected AI assistant workflow for automation, user support, memory, and operational tasks across multiple use cases.
Tools Used
Python, Linux, Docker, Telegram integration, OpenAI, Gemini, workflow design, profile isolation, and knowledge handling.
Security Consideration
Kept profiles separated, used controlled permissions, avoided exposing private data, and treated the assistant like an operational tool, not a toy.
Outcome
A practical AI infrastructure project that shows automation thinking, systems integration, and deployment discipline.
What I Learned
How to make AI useful inside a real workflow, how to manage boundaries, and how to build a more dependable assistant architecture.
